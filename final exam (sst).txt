 ans 1 get:
      1. In GET method, values are visible in the URL.	
      2. GET has a limitation on the length of the values, generally 255 characters.	
      3. This method supports only string data types.	
      4. GET Parameters remain in web browser history.	
 post:
      1. In POST method, values are not visible in the URL.
      2. POST has no limitation on the length of the values since they are submitted via the body of HTTP.
      3. This method supports different data types, such as string, numeric, binary, etc.
      4. Parameters are not saved in web browser history.
